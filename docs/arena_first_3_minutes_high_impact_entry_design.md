# Arena First 3 Minutes – High-Impact Entry Design

Dokumen ini mendefinisikan **desain 3 menit pertama Arena** yang bertujuan menciptakan *aha moment* cepat dengan **biaya kognitif rendah** namun **impact psikologis tinggi**.

Fokus dokumen ini adalah **urutan pengalaman (experience sequence)**, bukan filosofi atau engine detail.

---

## 1. Tujuan Desain

Dalam 180 detik pertama, user harus:

1. Merasa **tertampar** (menyadari blind spot)
2. Merasa **menyelesaikan sesuatu yang nyata**
3. Merasa **ini berbeda dari edtech lain**
4. Belum merasa lelah meskipun masalah terasa berat

Jika salah satu gagal, Arena entry dianggap gagal.

---

## 2. Prinsip Utama

- Jangan mulai dengan mengetik
- Mulai dengan **aksi berbiaya rendah** (klik / pilih)
- Tampilkan **konsekuensi langsung**
- Pastikan ada **micro-completion** di ≤90 detik

---

## 3. Experience Sequence (First 180 Seconds)

### 3.1 Screen 1 – Situation Drop (0–20s)

**Tujuan:** memberikan konteks tanpa analisis.

- 3–5 baris situasi konkret
- Tidak ada pertanyaan
- Tidak ada input

User hanya membaca dan memahami posisi dirinya.

---

### 3.2 Screen 2 – Forced Choice (20–45s)

**Tujuan:** memaksa aksi tanpa berpikir panjang.

- 2–3 opsi konkret
- Semua opsi tampak masuk akal
- Tidak ada opsi aman sempurna

User **harus memilih satu**.

---

### 3.3 Screen 3 – Decision Lock (45–60s)

**Tujuan:** menciptakan rasa tindakan nyata.

- Tombol eksplisit: **KUNCI KEPUTUSAN**
- Tidak bisa undo
- Visual konfirmasi singkat

Ini adalah **micro-win pertama**.

---

### 3.4 Screen 4 – Consequence Reveal (60–90s)

**Tujuan:** tamparan pertama.

- 1–2 konsekuensi langsung
- Salah satu hal penting hilang / terkompromi
- Bahasa singkat dan tegas

User menyadari:
> “Pilihan gue ada biayanya.”

---

### 3.5 Screen 5 – Status Update (90–120s)

**Tujuan:** memberikan rasa progres.

- Status berpikir muncul (mis. *forming / unstable*)
- Tanpa angka
- Tanpa skor

User merasa:
> “Ini bergerak.”

---

### 3.6 Screen 6 – First Reflection Prompt (120–180s)

**Tujuan:** refleksi ringan, bukan esai.

- Pertanyaan spesifik tentang pilihan tadi
- Jawaban pendek

Ini **pertama kalinya mengetik**.

---

## 4. Kenapa Urutan Ini Efektif

- Tamparan datang dari **konsekuensi**, bukan ceramah
- Refleksi datang **setelah aksi**, bukan sebelum
- Otak masuk flow tanpa sadar

---

## 5. Anti-Pattern (Dilarang)

- Pertanyaan panjang di awal
- Meminta justifikasi sebelum pilihan
- Menyuruh user menjelaskan strategi di menit awal

---

## 6. Outcome yang Diharapkan

Setelah 3 menit:

- User belum capek
- User penasaran
- User merasa “oh, ini beda”
- User siap masuk Arena utama

---

Dokumen ini dimaksudkan untuk dieksekusi langsung oleh UX designer dan AI coding agent sebagai **entry flow Arena**.
