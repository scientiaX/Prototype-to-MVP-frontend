# Arena Experience Layer – Intuitive High-Impact UX

Dokumen ini mendefinisikan **Experience Layer Arena**: bagaimana Arena *dirasakan* user — seru, tegang, intuitif — tanpa menambah kompleksitas atau mengganggu core logic & pedagogi.

Prinsip utama: **minimal actions, sequential focus, felt progress**.

---

## 1. Experience Goals

- User langsung paham *apa yang harus dilakukan*
- Satu layar = satu keputusan
- Tidak ada cognitive overload
- Progress terasa setiap 30–90 detik
- Friksi kecil tapi bermakna (bukan teknis)

---

## 2. Sequential Screen Model (No Clutter)

Arena **tidak menampilkan semua kontrol sekaligus**.

### Flow:
1. Situation screen
2. Action screen
3. Feedback micro screen
4. Next action

User hanya fokus ke **1 hal** per layar.

---

## 3. Visual States (Low-Fidelity, High-Signal)

### 3.1 Calm
- Warna netral
- Copy tenang
- Tidak ada animasi agresif

### 3.2 Focused
- Kontras meningkat
- Highlight area aktif
- Timer subtle

### 3.3 Urgent
- Accent color berubah
- Copy pendek & langsung
- Micro pulse animation

### 3.4 Critical
- Minimal UI
- Countdown jelas
- Bahasa AI tegas

Visual state **hanya refleksi state Arena**, bukan driver keputusan.

---

## 4. Micro Interaction Design

### 4.1 Commit Interaction
- Satu input field
- Tombol besar: **KUNCI KEPUTUSAN**
- Subtle haptic / visual confirm

### 4.2 Patch Interaction
- Highlight bagian yang boleh diubah
- Non-editable text di-dim

### 4.3 Choice Interaction
- 2–3 kartu pilihan
- Hover / tap feedback ringan

### 4.4 Task Interaction
- Task muncul full screen
- Instruksi 1 kalimat
- Tombol: **SELESAI TASK**

---

## 5. Micro Progress Indicators

Progress **tidak ditampilkan sebagai angka besar**.

Contoh:
- Status chip: `forming → consistent`
- Progress ring tipis
- Subtle text: "Decision stabilized"

Update terjadi **real-time**, tanpa menghentikan flow.

---

## 6. Micro Feedback (Kill-Style, Minimal)

- Toast singkat (1–2 detik)
- Ikon kecil + teks 2–3 kata
- Tidak menutup layar

Contoh:
- `Assumption found`
- `Trade-off locked`
- `Reasoning improved`

---

## 7. Language & Tone System

AI menggunakan **tone adaptif**:

- Calm: reflektif
- Focused: langsung
- Urgent: singkat & tegas

Tidak ada persona eksplisit.

---

## 8. Friction Design (Intentional, Minimal)

Friction digunakan untuk:
- Memaksa commit
- Menghentikan over-analysis

Bentuk:
- Extra tap untuk lock
- Short countdown
- Confirmation screen

Tidak ada:
- Form panjang
- Pengaturan teknis
- Multi-panel kompleks

---

## 9. One-Minute Wow Moment

Dalam ≤60 detik pertama:
- User sudah mengunci keputusan pertama
- Muncul 1–2 micro feedback
- Progress indicator bergerak

User merasa:
> "Oh, ini cepat, tapi serius."

---

## 10. Guardrails

Experience Layer **tidak boleh**:
- Menambah state baru
- Mengubah logic Arena
- Mengarahkan keputusan user

Hanya memvisualisasikan & memperkuat apa yang sudah terjadi.

---

Dokumen ini menjadi **referensi UX/UI Arena** dan dapat digunakan untuk desain visual, motion, dan copywriting tanpa mengubah core Arena Engine.
